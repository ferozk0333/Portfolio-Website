FinSight: Financial Research Assistant with LangChain and OpenAI

GitHub

All code is available on my GitHub.

Problem Statement

Investors and financial analysts face the challenge of sifting through vast amounts of financial information from multiple sources such as news articles, quarterly reports, and industry updates to make informed decisions. This process is time-consuming, error-prone, and inefficient due to repetitive tasks like manual aggregation and data retrieval. Additionally, traditional LLMs have limitations, such as token limits and high costs, when processing large volumes of text.

Proposed Solution

This app streamlines financial research by automating retrieval and analysis. It aggregates data, stores it in a vector database, and answers queries by retrieving and summarizing relevant chunks. By leveraging LLMs and efficient chunking, FinSight minimizes costs and improves accuracy.

It uses an Unstructured URL Loader to extract content and metadata. Text is split into chunks using a Recursive Character Text Splitter for optimal token usage. These chunks are converted to vector embeddings with OpenAI or HuggingFace and stored in FAISS for efficient searches.

FinSight retrieves relevant chunks via semantic search, combines them into a query prompt, and processes it with the LLM using RetrievalQAWithSourcesChain to generate accurate responses.

Business Domain

Financial Research, Investment Analysis.

The primary focus is on enabling efficient analysis of company metrics (e.g., EBITDA, PE ratios) and assisting in informed decision-making for stock evaluations and portfolio management.

Tech Stack

  • Frontend: Streamlit
  • Backend: Python, LangChain
  • Database: FAISS (In-memory Vector DB)
  • LLM Integration: OpenAI APIs
